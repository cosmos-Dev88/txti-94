How to make a web page with txti

The only thing you have to give txti is the "content". Txti takes care of the rest,but you
can take advantage of these option:

Custom URL

This is the part goes after"txti.es/"like"txti.es/barry". Currently only accepts
Lowercase letters(0-9),and dashes(-). If you put anything else in here will change it.


Custom Edit Code

Txti will give you a random edit code,but you can change it. You have to have this to
make changes to the txti,and it can never ever be retrieved if lost.


Title

Give your txti a relevant title so it shows up better when shared on social media
search results and in browser window


Author

Lets people know who made this!This shows up in some searches and social media
posts and your twitter handle work really nicely if you post the link on twitter.


Description

This is a short(200 character)ysummary of the page.this shows up in twitter cards and search results.Txti as a
description if you don't provide one


Make your txti easy to read and understand

You can make your txti robust and full-featured with links,images,lists, heading and
more. Txti uses a popular set of rules called Markdown


  • #Heading 1=biggest heading

    • ##Heading 2=second biggest heading and so on

    ###Heading 3=third biggest heading and so on

  • Two spaces(spaces bars) at the end of a line starts a new line
  • Return twice(leaving an empty line) starts a new paragraph
  • *itallics*=italics
  • **bold**=bold
  • line:{lind to txti}(http://.txti.es)=link to txti(note.be sure to include the link to txti part of the link)

  • image*:!{Monkey selfie} (http://i.imgur.com/FXSBf8c.jpg)
  • Bulleted lists:

    • -Bulleted item a

    -Bulleted item b

    -Bulleted item c

  • Nunber lists:

    • 2.Number item 2

    3.Number item 3

    *Images must be upload somewhere else{we recommend imazgur.com}because
    images can really show down page and we're all about"fast web pages for everybody,"
    images are not displayed by default. The reader has the option displayed the images in
    the page or just viewing links to them. see an example txti.es/images
